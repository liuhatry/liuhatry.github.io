C++异常

tags: C++学习笔记

Summary:

throw()

func() throw()

它是函数提供者和使用者的一种君子协定，标明该函数不抛出任何异常。 之所以说是君子协定，是因为实际上内部实现是需要人肉确保。 所以，在使用该方法的时候，不必把它至于 try/catch 异常处理块中。

如果一个标明throw()的函数内部发生了throw：

如果内部直接throw something，编译器会发现并指出；
如果是内部调用了一个可能throw something的函数，编译器无法发现，运行时一旦这个内部的函数throw，程序会abort。 

func() throw(type)

会抛出某种异常 

func() throw(..)

可能是任何类型的异常 

func()

可能抛出任何类型异常

catch()

try{}
catch(except1&){}  // 接受except1
catch(except2&){}  // 接受except2
catch(...){} //接受所有异常

一般都写成引用（except1&），原因很简单，效率。
